Footwork is a fully featured, expressive, and easy to use frontend framework based on KnockoutJS.
  • Component based
  • Automatic resolution and binding of resources
  • Integrated router with named outlet support
  • Transition effect and animation system with lifecycle events
  • RESTful data management
  • Does not require a build system or module loader (but it works great with them)
  • Inter-module communication and state sharing
  • No external dependencies
  • Free, open source (MIT License)

Footwork is an expressive, component-based, fully-featured frontend javascript framework based on KnockoutJS. It is built around the MVVM software architecture pattern and aims to be highly extensible and configurable while at the same time providing sane conventions and a consistent easy to use API.

I can hear you already, another framework??!

There are a lot of other front-end frameworks out there, why use this one? Footwork provides a forward-looking approach to single-page-app development utilizing the excellent two-way binding, component, and other features of KnockoutJS. It provides an approach to application development that keeps a firm grounding in proven methodologies while providing features and architecture that help make app development fun.

Why not just use Knockout, or Knockout + X?

KnockoutJS is a great library, but its focus on UI binding, components, and observables leaves it lacking a lot when it comes to building a full single-page app. Footwork fills in those gaps and provides novel features that makes application creation fun and expressive while eliminating a lot of common boilerplate many other frameworks require.

...and while you can piece together something from various independent libraries, what you end up with will likely not be as cohesive, well tested, documented, or iterated on as Footwork is.

I prefer a different framework, why would I consider Footwork?

Then stick with it, if it works.

But if you are open to something new, Footwork provides a uniquely clean and expressive API which is different from its competitors. Focusing on the ever changing javascript landscape, providing a powerful and expressive API, and remaining idiomatic and compatible with Knockout and its various libraries is the goal of this project.

It's basically another take on javascript development, it may or may not be your cup of tea.
