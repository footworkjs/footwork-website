Footwork is a fully featured, expressive, and easy to use frontend framework based on KnockoutJS.
  • Custom elements/components and declarative view models
  • Automatic resolution and binding of resources
  • Integrated router with unlimited named outlet support
  • Transition effect and animation system
  • Does not require a build system or module loader (but it works great with them)
  • Inter-module communication and state sharing
  • No external dependencies
  • Free, open source (MIT License)

Why Footwork?

There are a lot of front-end frameworks out there, why use Footwork? In short, it fills a niche for fans of KnockoutJS that want to use it to write fully single-page applications. It has other reasons to be used, such as its more novel features (ex: broadcastable / receivable) as well as its very expressive and easy to manage development methodology based on context and nesting.

KnockoutJS is a great library, but it lacks a lot when it comes to building a full single-page app. Footwork fills in those gaps and provides novel features that makes application creation fun and expressive.

If you want a forward-thinking but not a 'lets redo everything about everything' no-nonsense framework built around getting your application completed without a lot of fuss - then Footwork is worth checking out.
